## 为什么需要超越 JSVMP

JavaScript VMP（虚拟机保护）等方案把源逻辑编译成自定义字节码，再在解释器里执行。攻击者仍常能：

- 转储内存中的字节码与解释器
- 挂钩或替换运行时
- 跟踪执行轨迹
- 对比多次运行（轨迹往往高度可复现）

JSVMP、WASM 平坦化等路径共享一个**工程上常见的前提**：程序以相对固定、可复制的形态出现在客户端。本文 6 个方向尝试用不同理论动机**削弱或改写**该前提——有的只在真硬件假设下成立，有的依赖在线服务，有的只是布局/热图层面的启发式。

## 共同的假设

传统客户端保护常隐含：

```text
程序 = 客户端上可观察的固定制品
攻击者可以：静态分析、跟踪、对比、重放
多次执行 = 高度相似的字节码/轨迹
```

六方向各自「动」这一假设的方式不同：

| 方向 | 试图改变什么 |
|------|----------------|
| 量子启发混淆 | 真量子下：不可克隆/测量扰动；**本文实现是经典模拟** |
| 拓扑保护 | 把逻辑嵌进辫/纽结结构，抬高「从图还原逻辑」的结构难度 |
| 在线密钥流 | 无密钥时客户端只有密文；密钥按需下发（**PRF 实例，非真 OTP**） |
| 教学级 iO | 使等价电路的编码在计算上更难区分（**玩具参数**） |
| ZK 可验证执行 | 证明执行正确且限制对 **witness** 的泄露（**程序仍可在客户端**） |
| Local-Mixing | 打散门布局与热图相关性，抬高静态/模式匹配成本 |

## 量子电路混淆

**原型路径**: `MReverse/01_quantum_circuit_obfuscation/`

### 安全声明（先读）

| 层 | 含义 |
|----|------|
| 理论动机 | 真量子态：不可克隆、测量扰动 |
| **本文实现** | V8/Python **经典态向量模拟**（如 Float64Array） |
| 逆向含义 | 模拟器下攻击者可直接读取态数组；**不**享有真量子防护 |
| 攻击表中的 0% | 仅在「真量子硬件 + 对应威胁模型」叙事下有意义；模拟器场景应视为可读取 |

### 原理

把 Boolean 电路映射到量子门序列，并用相位门等变换改变态向量表示，而不改变测量概率分布——这是**教学上**展示「表示可变、功能不变」的路径。

### 编译路径

```text
Boolean 电路 (AND/OR/NOT/XOR)
    -> 量子门 (Toffoli/CNOT/X/H 等)
    -> 相位类门 (Z/S/T/RZ...)  // 改相位，测概率可保持
    -> 辅助比特 / 冗余
    -> 基变换
    -> 输出电路描述 (JSON 等)
```

### 数学基础（克制）

n 量子比特理想态占用 `2^n` 维复向量。酉变换保持范数。相位门可改变相对相位而不改变测量概率 `|alpha_i|^2`。  
**经典模拟代价**随 `2^n` 增长；实验仅适合很小的 n。

### 攻击示意（模拟器 vs 硬件）

| 攻击 | 硬件叙事（示意） | 经典模拟器 |
|------|------------------|------------|
| 中间态读取 | 测量破坏 | 可 dump 数组 |
| 克隆中间态 | 受不可克隆限制 | 可复制内存 |
| 门序列反推 | 表示不唯一 | 仍可动态执行/对比 |
| 黑盒 I/O | 与方案无关 | 与方案无关 |

仓库 `attack.py` 中的百分比是**作者定义的代理指标**，勿直接当成硬件安全结论。

### V8 映射

态向量用类型化数组模拟；门为稠密线性变换。内存与时间均对 n 敏感，仅适合演示规模。

### 局限

- 无真量子硬件时，**防护故事不能写满**
- 指数资源；与生产 JS 保护路径不同
- 价值主要在：换一套表示与攻击语言，而非上线替换 JSVMP

## 拓扑保护执行

**原型路径**: `MReverse/02_topological_protection/`

### 安全声明

辫群/纽结相关结构可提高「从静态图恢复源逻辑」的成本，但：

- **纽结等价判定**的复杂度结果不能直接翻译成「逆向不可能」
- 攻击者可**执行**程序、做 I/O 黑盒，不必计算 Jones 多项式
- 原型侧重 Boolean 电路，不是通用 JS 程序保护

### 原理

把逻辑编译为辫/纽结类结构；执行解释为沿结构求值（如 linking 等不变量路径）。Reidemeister 类局部变换用于混淆图外观而保持拓扑类型（在模型设定下）。

### 编译路径

```text
Boolean 电路
    -> 辫词 (生成元 sigma_i)
    -> 闭链 / 纽结图
    -> 局部拓扑型变换 (混淆外观)
    -> 冗余交叉 (不改变约定不变量)
    -> 可执行图结构 (JSON)
```

逻辑门到辫片段的映射是**约定式编码**（见目录 README），不是唯一标准。

### 数学基础（克制）

- 辫群：生成元与远交/辫关系
- Reidemeister 型局部操作：改变图示、保持拓扑类（在连续模型中）
- linking number 等：执行语义可用的标量
- Jones 等：计算难的不变量存在，**不等于**攻击者必须走这条路

### 攻击示意

静态读图、分解辫词、试图判定等价、黑盒 I/O 等在 `attack.py` 中有分项示意。动态执行往往比「证明两个纽结相同」更贴近真实逆向。

### V8 映射

图用 JSON；执行多为组合步，避免 `2^n` 态向量。可视化（ASCII/Canvas）是双刃剑：便于教学，也便于人眼跟读。

### 局限

仅 Boolean；无循环/字符串/通用控制流。适合小规模校验类逻辑实验，不适合「整站 JS 保护」叙事。

## 在线密钥流保护

**原型路径**: `MReverse/03_information_theoretic/`

### 安全声明（P0）

| 说法 | 是否成立 |
|------|----------|
| 真随机、等长、一次一密的 OTP + `C = P xor K` | 信息论：`H(P\|C) = H(P)`（经典结论） |
| **本文 demo**：`K(i) = HMAC-SHA256(MK, session\|\|i)` | **PRF 密钥流**；安全依赖 PRF/HMAC 与密钥托管，是**计算安全**，不要标成 Shannon 完美保密 |
| 客户端静态只有密文 | 在密钥不下发、不泄露时，静态分析密文信息量低（工程上有意义） |
| 密钥在服务端、信道被控 | 可退化为密钥抽取 / TLS 端点攻击等 |

下文称 **在线密钥流**；「OTP」仅作「一次一密风格」的历史称呼，实现是 PRF。

### 原理

客户端不持有明文指令流；按序号向服务端取密钥材料，本地解密后执行，并可回报哈希以做轻量校验（具体协议以代码为准）。

### 架构

```text
  Server                         Client
  ------                         ------
  master key MK
       |                              |
       |  K(i) = PRF(MK, sid, i)      |
       |----------------------------->|  decrypt: P[i] = C[i] xor K(i)
       |                              |  execute P[i]
       |  optional H(K(i)) check      |
       |<-----------------------------|
       |                              |
       +---- repeat until done -------+
```

### 数学基础

- **真 OTP**：信息论安全需真随机 K、与 P 等长、不复用
- **PRF 流**：在 PRF 假设下模拟随机性；泄露 MK 或侧信道则整体失败
- 会话号 / 防重放：工程属性，需单独威胁模型

### 攻击示意

静态读密文、单步内存转储、密钥托管沦陷、重放、选择指令等，见 `attack.py`。示意均分约 12% **不能**解读为「已达信息论 12% 泄露」。

### V8 映射

密文 `Uint8Array`；密钥经 WebSocket/fetch；解密为 XOR；哈希可用 Web Crypto。开销接近线性，但**强依赖网络与可用性**。

### 局限

- 服务器单点；离线不可用
- RTT；需预取等工程优化
- 威胁模型必须包含「服务端与信道」

## 不可区分混淆 (iO)

**原型路径**: `MReverse/04_indistinguishability_obfuscation/`

### 安全声明

密码学 iO 是极强原语；**本仓库实现是 LWE 玩具电路演示**，参数（如小维度 n）**不提供**真实格密码安全。文中任何「不可区分」仅指 demo 内统计/结构实验，不是定理级 iO。

### 原理

直觉：功能等价的电路，经混淆后在计算上难以判断「来自哪一份源电路」。真实构造复杂且开销巨大；此处用 LWE 编码门与再随机化**模仿工作流**。

### 编译路径

```text
Boolean 电路
    -> 每门 LWE 式编码 (玩具参数)
    -> 再随机化 / 密钥切换类步骤 (示意)
    -> 可求值的编码电路
```

### 数学基础（克制）

LWE：带噪声线性样本上恢复秘密被认为难（渐近）。玩具 n 上 BKZ 等具体数字**不要**写成生产安全边界。近年候选 iO 构造若提及，需可核查文献；否则只写「实用化仍远」。

### 攻击示意

密钥恢复、区分实验、黑盒 I/O、格算法玩具实例等——均在 demo 参数下解释。

### V8 映射

整数数组模 q；矩阵乘加。复杂度随电路与维数上升。

### 局限

教育向；与「上线保护业务 JS」不是同一问题。

## 零知识可验证执行

**原型路径**: `MReverse/05_zk_wasm/`

### 安全声明（P0）

**选定叙事（与常见 ZK 一致）**：

- 客户端**可以持有并执行**程序（WASM/字节码）
- ZK 证明的是「在某公共输入下执行满足约束」，并限制 **witness / 私有输入** 泄露
- **不是**「程序本体自动从客户端消失」
- 本仓库为教学级 Sigma/电路，**不是**生产 zkSNARK / 商业 zkWASM 栈；勿宣称 128-bit 安全除非给出完整参数与证明系统

若未来改为「仅服务端持有程序」，须另开一篇架构，与当前目录实现区分。

### 原理

```text
  Prover (client)                    Verifier (server)
  ---------------                    -----------------
  program + public_in + witness
       | execute / trace
       v
  constraints (toy)
       | prove
       v
  (public_out, proof) -------------> verify(proof)
                                      accept / reject
```

### 数学基础（克制）

完备性、可靠性、零知识是证明系统性质；Fiat-Shamir 把交互变非交互时依赖哈希模型。玩具实现满足「能跑通演示」，不等于审计级证明系统。

### 攻击示意

伪造证明、提取 witness、侧信道、验证者逻辑错误等——以 `attack.py` 为准，百分比仍为示意。

### V8 映射

执行轨迹、哈希承诺、模运算响应等；与「把业务逻辑藏进客户端却声称别人看不见代码」无关。

### 局限

适合**可验证计算 / 私有输入**场景；对「防逆向源码」只有间接帮助（例如配合服务端策略），不能单独等同代码隐藏。

## Local-Mixing 热图消相关

**原型路径**: `MReverse/06_local_mixing/`

### 安全声明

- 通过打散门布局、插入无效计算、等价改写等，降低静态局部性与热图相关性
- 「热图消相关」是**经验度量**，不是 Shannon 意义的信息论安全
- 名称若对应外部论文，应给引用；否则视为**实验命名**，避免未经验证的专有名词光环
- 代码仍可被执行与动态分析；定位是 **VMP/布局增强**，不是密码学终态

### 原理

```text
紧密布局:   [AND][AND][XOR]          热图高峰聚集
混合后:     [XOR] ... [AND] ... [AND]  峰值打散 (示意)
```

### 变换

依赖 `mixer.py`：重排、填充、等价替换、无用链等。具体列表以目录 README 为准，本文不重复维护第二份清单。

### 攻击示意

模式匹配、热图聚类、动态 trace 对齐等。示意分约 24% 表示「在作者脚本下仍有可见结构」，不是「已防住 76% 的全世界攻击者」。

### V8 映射

普通运算与控制流；膨胀可很大（数量级依赖填充策略）。可与现有 VMP 管道叠加。

### 局限

可解码；强动态分析与 I/O 黑盒仍有效。价值在工程可部署性，不在可证明性。

## 多维度对比矩阵

| 前向安全 | N | N | 会话设计相关 | 参数相关 | 会话相关 | N |

| 场景 | 更合理的方向 |
|------|----------------|
| 立即增强现有 VMP | Local-Mixing |
| 高价值片段 + 可接受在线 | 密钥流 |
| 可验证计算 / 私有输入 | ZK（真生产请换成熟证明栈） |
| 教学演示表示变换 | 量子模拟 / 拓扑 / 玩具 iO |

## 混合方案建议

### 方案 A：VMP + Local-Mixing（偏实用）

在现有字节码/VMP 管道上增加布局打散与填充。无密码学银弹，但可落地。

### 方案 B：密钥流 + 可验证执行（偏高价值在线）

高价值指令在线解密；关键计算可附带证明。运维与延迟成本高。

### 方案 C：拓扑或玩具 iO（偏研究）

适合论文式演示与小型电路，不作「全站保护」承诺。

### 方案 D：分层（示意）

```text
  UI / 非敏感  -------->  常规构建
  中敏感逻辑  -------->  VMP + Local-Mixing
  高敏感片段  -------->  在线密钥流 (+ 可选证明)
  研究实验    -------->  01/02/04 目录独立维护
```

## 局限与未来方向

### 各方案关键局限

| 方向 | 关键局限 |
|------|----------|
| 量子 | 经典模拟无硬件防护；指数开销 |
| 拓扑 | Boolean；动态攻击绕过「等价判定」叙事 |
| 密钥流 | 网络与服务器信任 |
| iO | 玩具参数；真 iO 不实用 |
| ZK | 不自动隐藏程序；玩具证明系统 |
| Local-Mixing | 可解码；动态分析仍强 |

### 未来方向（研究向）

张量网络等量子启发经典算法、拓扑扩展控制流、密钥流预取、真实 FHE/证明库对接、动态垃圾与抗 LLM 去混淆评测等——均需单独实验，本文不展开承诺。

### 开放问题

- 组合时开销与安全是否可加
- 对 LLM 辅助去混淆的抵抗如何度量
- 与 TEE 等硬件路径如何分工
- 可证明混淆的代价下界

## 结论

传统 JSVMP 类方案常共享「客户端固定制品」这一工程前提。MReverse 六方向分别从表示、结构、在线密钥、弱 iO 工作流、可验证执行与布局热图切入，**改写或削弱**该前提——但多数在本文实现层级上仍是**原型与示意**。

对「今天能叠在工程上的」：优先 **Local-Mixing 增强现有 VMP**，高价值路径慎用 **在线密钥流** 并写清威胁模型。对「写进论文/课堂的」：量子模拟、拓扑、玩具 iO、玩具 ZK 更合适，且必须保留安全声明。
